# IWordBreaker::BreakText


## -description


<p class="CCE_Message">[Indexing Service is no longer supported as of Windows XP and is unavailable for use as of Windows 8. Instead, use <a href="https://msdn.microsoft.com/6da601c6-3742-40ad-99f2-8817f7f642b3">Windows Search</a> for client side search and  <a href=" http://go.microsoft.com/fwlink/p/?linkid=258445">Microsoft Search Server Express</a> for server side search.]

Parses text to identify words and phrases and provides the results to the WordSink and PhraseSink objects.


## -parameters




### -param pTextSource [in]

A pointer to a <a href="https://msdn.microsoft.com/f56cf8b2-cd4c-43d7-935f-d8edaa8705fd">TEXT_SOURCE</a> structure that contains Unicode text. 



### -param pWordSink [in]

A pointer to the WordSink object that receives and handles words generated by this method. <b>NULL</b> indicates that this method identifies phrases only. 


### -param pPhraseSink [in]

A pointer to the PhraseSink object that receives and handles phrases generated by this method. <b>NULL</b> indicates that this method identifies individual words, not phrases.

## -remarks



This method should be optimized for <i>pPhraseSink</i> = 0 because word breakers more commonly parse for individual words than phrases. Either <i>pWordSink</i> or <i>pPhraseSink</i> may be <b>NULL</b>, but not both. 



Use <b>pfnFillTextBuffer</b>, the function pointer element in the <a href="https://msdn.microsoft.com/f56cf8b2-cd4c-43d7-935f-d8edaa8705fd">TEXT_SOURCE</a> structure, to replenish the source text. The <b>BreakText</b> method must handle all <b>pfnFillTextBuffer</b> return values. If an error occurs, finish processing the text in the buffer before handling the error.
