# _D3DRENDERSTATETYPE enumeration


## -description


Render states define set-up states for all kinds of vertex and pixel processing. Some render states set up vertex processing, and some set up pixel processing (see <a href="https://msdn.microsoft.com/8ed8a2c3-f88e-427a-9b92-dc244dd453c9">Render States (Direct3D 9)</a>). Render states can be saved and restored using stateblocks (see <a href="https://msdn.microsoft.com/6b1917a8-8685-40c3-983d-6bd2fed95642">State Blocks Save and Restore State (Direct3D 9)</a>).


## -enum-fields




### -field D3DRS_ZENABLE

Depth-buffering state as one member of the <a href="https://msdn.microsoft.com/5e5ce48b-8859-43e0-a9b4-5972cf6bf781">D3DZBUFFERTYPE</a> enumerated type. Set this state to D3DZB_TRUE to enable z-buffering, D3DZB_USEW to enable w-buffering, or D3DZB_FALSE to disable depth buffering. 

The default value for this render state is D3DZB_TRUE if a depth stencil was created along with the swap chain by setting the 
    EnableAutoDepthStencil member of the <a href="https://msdn.microsoft.com/d677aeb7-a188-4ddc-b8c9-48e13676e9c8">D3DPRESENT_PARAMETERS</a> structure to <b>TRUE</b>, and D3DZB_FALSE otherwise.


### -field D3DRS_FILLMODE

One or more members of the <a href="https://msdn.microsoft.com/be835432-e8d5-4afb-a810-2dac25bdc9dc">D3DFILLMODE</a> enumerated type. The default value is D3DFILL_SOLID. 


### -field D3DRS_SHADEMODE

One or more members of the <a href="https://msdn.microsoft.com/ba4e0c62-b496-427b-a324-2fb560d153ba">D3DSHADEMODE</a> enumerated type. The default value is D3DSHADE_GOURAUD. 


### -field D3DRS_ZWRITEENABLE

<b>TRUE</b> to enable the application to write to the depth buffer. The default value is <b>TRUE</b>. This member enables an application to prevent the system from updating the depth buffer with new depth values. If <b>FALSE</b>, depth comparisons are still made according to the render state D3DRS_ZFUNC, assuming that depth buffering is taking place, but depth values are not written to the buffer. 


### -field D3DRS_ALPHATESTENABLE

<b>TRUE</b> to enable per pixel alpha testing. If the test passes, the pixel is processed by the frame buffer. Otherwise, all frame-buffer processing is skipped for the pixel. 

The test is done by comparing the incoming alpha value with the reference alpha value, using the comparison function provided by the D3DRS_ALPHAFUNC render state. The reference alpha value is determined by the value set for D3DRS_ALPHAREF. For more information, see <a href="https://msdn.microsoft.com/368c3d12-2c8b-43e3-94c3-bccfe6c73e66">Alpha Testing State (Direct3D 9)</a>.

 The default value of this parameter is <b>FALSE</b>.


### -field D3DRS_LASTPIXEL

The default value is <b>TRUE</b>, which enables drawing of the last pixel in a line. To prevent drawing of the last pixel, set this value to <b>FALSE</b>. For more information, see <a href="https://msdn.microsoft.com/8a7ed4b1-25a1-4984-baea-6e176f0545ea">Outline and Fill State (Direct3D 9)</a>.


### -field D3DRS_SRCBLEND

One member of the <a href="https://msdn.microsoft.com/60ff384c-15a0-4c6f-9e2c-59fdea76b7a1">D3DBLEND</a> enumerated type. The default value is D3DBLEND_ONE. 


### -field D3DRS_DESTBLEND

One member of the <a href="https://msdn.microsoft.com/60ff384c-15a0-4c6f-9e2c-59fdea76b7a1">D3DBLEND</a> enumerated type. The default value is D3DBLEND_ZERO. 


### -field D3DRS_CULLMODE

Specifies how back-facing triangles are culled, if at all. This can be set to one member of the <a href="https://msdn.microsoft.com/b669307c-0d40-4ecb-8a2e-8bd1d9c65647">D3DCULL</a> enumerated type. The default value is D3DCULL_CCW. 


### -field D3DRS_ZFUNC

One member of the <a href="https://msdn.microsoft.com/999af3eb-a208-4312-acee-373192ea69e4">D3DCMPFUNC</a> enumerated type. The default value is D3DCMP_LESSEQUAL. This member enables an application to accept or reject a pixel, based on its distance from the camera. 

The depth value of the pixel is compared with the depth-buffer value. If the depth value of the pixel passes the comparison function, the pixel is written.

The depth value is written to the depth buffer only if the render state is <b>TRUE</b>.

Software rasterizers and many hardware accelerators work faster if the depth test fails, because there is no need to filter and modulate the texture if the pixel is not going to be rendered.


### -field D3DRS_ALPHAREF

Value that specifies a reference alpha value against which pixels are tested when alpha testing is enabled. This is an 8-bit value placed in the low 8 bits of the DWORD render-state value. Values can range from 0x00000000 through 0x000000FF. The default value is 0.


### -field D3DRS_ALPHAFUNC

One member of the <a href="https://msdn.microsoft.com/999af3eb-a208-4312-acee-373192ea69e4">D3DCMPFUNC</a> enumerated type. The default value is D3DCMP_ALWAYS. This member enables an application to accept or reject a pixel, based on its alpha value. 


### -field D3DRS_DITHERENABLE

<b>TRUE</b> to enable dithering. The default value is <b>FALSE</b>. 


### -field D3DRS_ALPHABLENDENABLE

<b>TRUE</b> to enable alpha-blended transparency. The default value is <b>FALSE</b>. 

The type of alpha blending is determined by the D3DRS_SRCBLEND and D3DRS_DESTBLEND render states.


### -field D3DRS_FOGENABLE

<b>TRUE</b> to enable fog blending. The default value is <b>FALSE</b>. For more information about using fog blending, see <a href="https://msdn.microsoft.com/library/windows/hardware/hh997897">Fog</a>. 


### -field D3DRS_SPECULARENABLE

<b>TRUE</b> to enable specular highlights. The default value is <b>FALSE</b>. 

Specular highlights are calculated as though every vertex in the object being lit is at the object's origin. This gives the expected results as long as the object is modeled around the origin and the distance from the light to the object is relatively large. In other cases, the results as undefined.

When this member is set to <b>TRUE</b>, the specular color is added to the base color after the texture cascade but before alpha blending.


### -field D3DRS_FOGCOLOR

Value whose type is <a href="https://msdn.microsoft.com/a4425774-fd4b-4f5c-9e10-7679bc2795f6">D3DCOLOR</a>. The default value is 0. For more information about fog color, see <a href="https://msdn.microsoft.com/76366496-553d-4dbf-868d-d58b5377d36a">Fog Color (Direct3D 9)</a>. 


### -field D3DRS_FOGTABLEMODE

The fog formula to be used for pixel fog. Set to one of the members of the <a href="https://msdn.microsoft.com/cd83c914-bc1d-4f66-b5a6-7984b7ec52cd">D3DFOGMODE</a> enumerated type. The default value is D3DFOG_NONE. For more information about pixel fog, see <a href="https://msdn.microsoft.com/6fcfb9fa-cacc-4dbc-bfc6-85d533dbfbf8">Pixel Fog (Direct3D 9)</a>. 


### -field D3DRS_FOGSTART

Depth at which pixel or vertex fog effects begin for linear fog mode. The default value is 0.0f. Depth is specified in world space for vertex fog and either device space [0.0, 1.0] or world space for pixel fog. For pixel fog, these values are in device space when the system uses z for fog calculations and world-world space when the system is using eye-relative fog (w-fog). For more information, see <a href="https://msdn.microsoft.com/a3c5f439-6937-4ba9-991f-5a4154447cf9">Fog Parameters (Direct3D 9)</a> and <a href="https://msdn.microsoft.com/6fcfb9fa-cacc-4dbc-bfc6-85d533dbfbf8">Eye-Relative vs. Z-based Depth</a>. 

Values for this render state are floating-point values. Because the <a href="https://msdn.microsoft.com/65738aae-aa90-48c5-8c9c-1927d1c92c54">IDirect3DDevice9::SetRenderState</a> method accepts 
    DWORD values, your application must cast a variable that contains the value, as shown in the following code example.

### -field D3DRS_RANGEFOGENABLE

<b>TRUE</b> to enable range-based vertex fog. The default value is <b>FALSE</b>, in which case the system uses depth-based fog. In range-based fog, the distance of an object from the viewer is used to compute fog effects, not the depth of the object (that is, the z-coordinate) in the scene. In range-based fog, all fog methods work as usual, except that they use range instead of depth in the computations. 

Range is the correct factor to use for fog computations, but depth is commonly used instead because range is time-consuming to compute and depth is generally already available. Using depth to calculate fog has the undesirable effect of having the fogginess of peripheral objects change as the viewer's eye moves - in this case, the depth changes and the range remains constant.

Because no hardware currently supports per-pixel range-based fog, range correction is offered only for vertex fog.

For more information, see <a href="https://msdn.microsoft.com/76989eb3-cd95-4dfc-ba0f-7563860b531c">Vertex Fog (Direct3D 9)</a>.


### -field D3DRS_STENCILENABLE

<b>TRUE</b> to enable stenciling, or <b>FALSE</b> to disable stenciling. The default value is <b>FALSE</b>. For more information, see <a href="https://msdn.microsoft.com/984f0a98-4a74-44c3-a9d0-f5d3f12f7e4e">Stencil Buffer Techniques (Direct3D 9)</a>.


### -field D3DRS_STENCILFAIL

Stencil operation to perform if the stencil test fails. Values are from the <a href="https://msdn.microsoft.com/338a3b7f-236a-484f-96b8-1e6bfb014484">D3DSTENCILOP</a> enumerated type. The default value is D3DSTENCILOP_KEEP.


### -field D3DRS_STENCILZFAIL

Stencil operation to perform if the stencil test passes and the depth test (z-test) fails. Values are from the <a href="https://msdn.microsoft.com/338a3b7f-236a-484f-96b8-1e6bfb014484">D3DSTENCILOP</a> enumerated type. The default value is D3DSTENCILOP_KEEP.


### -field D3DRS_STENCILPASS

Stencil operation to perform if both the stencil and the depth (z) tests pass. Values are from the <a href="https://msdn.microsoft.com/338a3b7f-236a-484f-96b8-1e6bfb014484">D3DSTENCILOP</a> enumerated type. The default value is D3DSTENCILOP_KEEP.


### -field D3DRS_STENCILFUNC

Comparison function for the stencil test. Values are from the <a href="https://msdn.microsoft.com/999af3eb-a208-4312-acee-373192ea69e4">D3DCMPFUNC</a> enumerated type. The default value is D3DCMP_ALWAYS. 

The comparison function is used to compare the reference value to a stencil buffer entry. This comparison applies only to the bits in the reference value and stencil buffer entry that are set in the stencil mask (set by the D3DRS_STENCILMASK render state). If <b>TRUE</b>, the stencil test passes.


### -field D3DRS_STENCILREF

An int reference value for the stencil test. The default value is 0.


### -field D3DRS_STENCILMASK

Mask applied to the reference value and each stencil buffer entry to determine the significant bits for the stencil test. The default mask is 0xFFFFFFFF. 


### -field D3DRS_STENCILWRITEMASK

Write mask applied to values written into the stencil buffer. The default mask is 0xFFFFFFFF. 


### -field D3DRS_TEXTUREFACTOR

Color used for multiple-texture blending with the D3DTA_TFACTOR texture-blending argument or the D3DTOP_BLENDFACTORALPHA texture-blending operation. The associated value is a <a href="https://msdn.microsoft.com/a4425774-fd4b-4f5c-9e10-7679bc2795f6">D3DCOLOR</a> variable. The default value is opaque white (0xFFFFFFFF).


### -field D3DRS_WRAP0

Texture-wrapping behavior for multiple sets of texture coordinates. Valid values for this render state can be any combination of the D3DWRAPCOORD_0 (or D3DWRAP_U), D3DWRAPCOORD_1 (or D3DWRAP_V), D3DWRAPCOORD_2 (or D3DWRAP_W), and D3DWRAPCOORD_3 flags. ### -field D3DRS_MULTISAMPLEANTIALIAS


    bool value that determines how individual samples are computed when using a multisample render-target buffer. ### -field D3DRS_MULTISAMPLEMASK

Each bit in this mask, starting at the least significant bit (LSB), controls modification of one of the samples in a multisample render target. The default value is 0xFFFFFFFF. 

This render state enables use of a multisample buffer as an accumulation buffer, doing multipass rendering of geometry where each pass updates a subset of samples.

If there are n multisamples and k enabled samples, the resulting intensity of the rendered image should be k/n. Each component RGB of every pixel is factored by k/n.

The type of separate alpha blending is determined by the D3DRS_SRCBLENDALPHA and D3DRS_DESTBLENDALPHA render states.


### -field D3DRS_SRCBLENDALPHA

One member of the <a href="https://msdn.microsoft.com/60ff384c-15a0-4c6f-9e2c-59fdea76b7a1">D3DBLEND</a> enumerated type.  This value is ignored unless D3DRS_SEPARATEALPHABLENDENABLE is <b>TRUE</b>.  The default value is D3DBLEND_ONE.


### -field D3DRS_DESTBLENDALPHA

One member of the <a href="https://msdn.microsoft.com/60ff384c-15a0-4c6f-9e2c-59fdea76b7a1">D3DBLEND</a> enumerated type.  This value is ignored unless D3DRS_SEPARATEALPHABLENDENABLE is <b>TRUE</b>.  The default value is D3DBLEND_ZERO.


### -field D3DRS_BLENDOPALPHA

Value used to select the arithmetic operation applied to separate alpha blending when the render state, D3DRS_SEPARATEALPHABLENDENABLE, is set to <b>TRUE</b>. 
    


Valid values are defined by the <a href="https://msdn.microsoft.com/ae750d84-ed7d-4a76-bf64-eae8828c66c7">D3DBLENDOP</a> enumerated type. The default value is D3DBLENDOP_ADD.

If the D3DPMISCCAPS_BLENDOP device capability is not supported, then D3DBLENDOP_ADD is performed. See <a href="https://msdn.microsoft.com/7912c682-c179-453b-8a34-e87958217500">D3DPMISCCAPS</a>.


### -field D3DRS_FORCE_DWORD

Forces this enumeration to compile to 32 bits in size. Without this value, some compilers would allow this enumeration to compile to a size other than 32 bits. This value is not used.
