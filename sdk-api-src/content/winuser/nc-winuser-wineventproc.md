# WINEVENTPROC callback function


## -description


An application-defined callback (or hook) function that the system calls in response to events generated by an accessible object. The hook function processes the event notifications as required. Clients install the hook function and request specific types of event notifications by calling <a href="https://msdn.microsoft.com/090bda1b-0635-4aa3-ae33-3987b36e30b8">SetWinEventHook</a>.

The <a href="wineventproc">WINEVENTPROC</a> type defines a pointer to this callback function. <i>WinEventProc</i> is a placeholder for the application-defined function name.


## -parameters




### -param hWinEventHook

Type: <b>HWINEVENTHOOK</b>

Handle to an event hook function. This value is returned by <a href="https://msdn.microsoft.com/090bda1b-0635-4aa3-ae33-3987b36e30b8">SetWinEventHook</a> when the hook function is installed and is specific to each instance of the hook function.


### -param event

Type: <b><a href="https://msdn.microsoft.com/4553cafc-450e-4493-a4d4-cb6e2f274d46">DWORD</a></b>

Specifies the event that occurred. This value is one of the <a href="https://msdn.microsoft.com/library/windows/hardware/ff597567">event constants</a>.


### -param hwnd

Type: <b><a href="https://msdn.microsoft.com/4553cafc-450e-4493-a4d4-cb6e2f274d46">HWND</a></b>

Handle to the window that generates the event, or <b>NULL</b> if no window is associated with the event. For example, the mouse pointer is not associated with a window.


### -param idObject

Type: <b><a href="https://msdn.microsoft.com/4553cafc-450e-4493-a4d4-cb6e2f274d46">LONG</a></b>

Identifies the object associated with the event. This is one of the <a href="https://msdn.microsoft.com/dc1603f8-29e5-4acd-817a-c6957feaff6c">object identifiers</a> or a custom object ID.


### -param idChild

Type: <b><a href="https://msdn.microsoft.com/4553cafc-450e-4493-a4d4-cb6e2f274d46">LONG</a></b>

Identifies whether the event was triggered by an object or a child element of the object. If this value is CHILDID_SELF, the event was triggered by the object; otherwise, this value is the child ID of the element that triggered the event.


### -param idEventThread


### -param dwmsEventTime

Type: <b><a href="https://msdn.microsoft.com/4553cafc-450e-4493-a4d4-cb6e2f274d46">DWORD</a></b>

Specifies the time, in milliseconds, that the event was generated.


#### - dwEventThread

Type: <b><a href="https://msdn.microsoft.com/4553cafc-450e-4493-a4d4-cb6e2f274d46">DWORD</a></b>

Identifies the thread that generated the event, or the thread that owns the current window.


## -returns



This callback function does not return a value.




## -remarks



Within the hook function, the parameters <i>hwnd</i>, <i>idObject</i>, and <i>idChild</i> are used when calling <a href="https://msdn.microsoft.com/d453c163-3918-4a1c-9636-16816227a295">AccessibleObjectFromEvent</a>.

Servers generate events by calling <a href="https://msdn.microsoft.com/08e74d45-95b6-44c2-a2e0-5ba6ffdcd56a">NotifyWinEvent</a>.

Create multiple callback functions to handle different events. For more information, see <a href="https://msdn.microsoft.com/7e999335-6a41-4d22-82ef-1a8dd6cb656e">Registering a Hook Function</a>.
